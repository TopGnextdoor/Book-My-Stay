# 🏨 Hotel Booking Management System  
### App-Based Learning Using Core Java & Data Structures  

---

## 📌 Overview  
The **Hotel Booking Management System** is a console-based application built using **Core Java and Data Structures** to simulate real-world hotel reservation workflows.  

This project demonstrates **how and why data structures are used in real applications**, bridging the gap between theory and practical system design.

---

## 🎯 Objective  
The primary goal of this project is to showcase how **Core Java concepts and fundamental data structures** can be applied to solve real-world software engineering problems such as:

- Managing booking requests efficiently  
- Maintaining real-time room availability  
- Preventing duplicate or conflicting bookings  
- Designing scalable and maintainable systems  

---

## 🚀 Key Features  

### ✔ Fair Request Handling  
Implements **FIFO (First-In-First-Out)** principles to ensure bookings are processed in order.

### ✔ Real-Time Inventory Management  
Ensures **consistent room availability** across all booking operations.

### ✔ Double-Booking Prevention  
Uses **uniqueness constraints** to avoid conflicts in reservations.

### ✔ Modular & Extensible Design  
Structured using **Object-Oriented Programming (OOP)** principles for easy scalability.

---

## 🧠 Learning Approach  

This project follows an **incremental learning model**, where each stage introduces new concepts while reinforcing previous ones.

Each implementation step focuses on:

- Why a specific data structure is used  
- What problem it solves in a real-world scenario  
- Limitations of simpler approaches  
- Refactoring towards better design

## ⚙️ How It Works  

1. User requests a booking  
2. System checks room availability  
3. Booking is processed using FIFO logic  
4. Data structures ensure:
   - No duplicate bookings  
   - Consistent room allocation  
5. Booking is confirmed and stored
